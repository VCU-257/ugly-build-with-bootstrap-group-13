## 1. My Assigned Work
I built the character flow for the project, including the following pages:

* CharacterCreation.html
* CharacterPage.html
* CharacterSuccess.html

My work focused on separating the character library view from the character builder, creating the form for entering character details, and adding the success screen that completes the flow after saving a character.

## 2. Bootstrap Implementation
For my assigned pages, I stayed with the Bootstrap-based plan and used standard components for layout, navigation, forms, and action buttons instead of building custom UI elements from scratch.

* **Components Used:** Bootstrap containers, grid rows and columns, nav pills, cards/panel sections, form controls, form selects, textareas, badges, and buttons

On CharacterCreation.html, I used Bootstrap form controls and the grid system to organize the character builder into clear sections like basics, abilities, combat, equipment, and background details. On CharacterPage.html and CharacterSuccess.html, I used card-style sections, buttons, and responsive layout utilities to keep the pages consistent with the rest of the app.

## 3. Technical Challenges & Solutions
The hardest part of my assigned work was organizing the character feature into multiple connected screens while keeping the layout responsive and easy to follow. I had to make sure the library page, creation page, and success page felt like one flow instead of three unrelated screens.

I solved that by separating responsibilities across the pages and using Bootstrap's grid and spacing utilities to keep each screen structured. I also reused the same navigation and visual pattern across the character pages so the user could move through the flow without confusion.
